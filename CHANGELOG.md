# Changelog

## [1.1.0.0] - 2026-07-09

### Added
- Calm Utilitarian 디자인 시스템 도입 (`DESIGN.md`): 차분한 블루/앰버 팔레트, Pretendard 폰트, 8px 기준 스페이싱
- 화면 중앙에 큰 카운트다운 숫자(Hero Number)로 도착까지 남은 시간을 표시
- 모니터링 중임을 알려주는 펄스 점과 "감시 중" 라벨, 시작/종료 버튼이 모니터링 상태에 따라 서로 바뀌어 표시
- 모니터링 시작 전 도착 정보를 먼저 확인해, 정보가 없으면 시작하지 않고 실패 팝업으로 안내

### Fixed
- 폴링 간격 사이 ETA가 여러 임계값(15/10/5/3/1분)을 한꺼번에 건너뛰면 중간 알림이 영구적으로 누락되던 버그 수정
- Android 12+ 기기에서 모니터링 시작 검증 중 앱이 백그라운드로 전환되면 포그라운드 서비스 시작이 거부되어 발생할 수 있던 크래시 방지
- 시작 실패 안내 팝업이 화면 전환/종료 이후에 표시되어 발생할 수 있던 창 누수 방지
- API 24-25 기기에서 Pretendard 폰트가 적용되지 않고 기본 폰트로 표시되던 문제 수정 (`app:fontFamily` 적용)
- 좁은 화면/큰 글꼴 설정에서 노선 라벨이 화면 밖으로 잘리던 문제 수정
- 도착 정보 조회/처리 실패 시 원인을 진단할 수 있도록 로그 기록 추가

## [1.0.0.0] - 2026-07-07

### Added
- 앱 버전을 `VERSION` 파일과 `computedVersionCode` 패턴으로 자동 관리하도록 변경 (기존 하드코딩된 versionCode=1 제거)
- 모니터링 시작 시 배터리 최적화 예외 요청 안내(다이얼로그 + 설정 화면 연결), 삼성 기기 절전 앱 관리 안내 문구 포함

### Fixed
- API 33 미만 기기에서 `URLEncoder.encode` 호출이 크래시하던 문제 수정
- "다음 차량"을 선택했을 때 1번 차량 데이터 유무만으로 "정보 없음"으로 오판하던 로직 수정
- 브로드캐스트 리시버 등록 시 EXPORTED/NOT_EXPORTED 플래그 누락 수정
- WakeLock에 타임아웃과 폴링 주기마다 갱신 로직을 추가하고, 백그라운드 스레드와 서비스 종료 사이의 경쟁 상태를 동기화로 제거
- 선택 차량 도착 정보가 일시적으로 없을 때 상단 알림이 갱신되지 않고 오래된 정보를 계속 보여주던 문제 수정
- 폴링 중 예상치 못한 런타임 예외가 발생하면 이후 모든 폴링이 조용히 영구 중단되던 위험 제거 (예외를 잡아 상태로 알림)

### Removed
- minSdk 24에서 항상 참인 죽은 `SDK_INT` 분기 정리
- 사용하지 않는 리소스(`R.color.black`, `R.color.white`, `R.string.monitor_started`) 제거
